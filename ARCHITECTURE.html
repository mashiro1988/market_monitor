④ 一次远程数据周期 remote_data_cycle（sequenceDiagram）

独立于 5min 扫描的 APScheduler job（默认 1h）。pull → 板块计算 → 写 DB 串行；若 sector_scan 失败或写出 0 行，会保留 pending 并在后续 cycle 重扫同一 cutoff，详细 file:line 见 ARCHITECTURE.md。
sequenceDiagram
    autonumber
    participant Sched as APScheduler
    participant Puller as remote_puller
cycle()
    participant FS as remote_fs
(SFTP)
    participant BMAC as BMAC 服务器
    participant SS as sector_scanner
    participant CMCdb as cmc_symbol_categories
    participant DB as sector_returns

    Sched->>Puller: run_remote_data_cycle()
    loop 每个 dataset (按 poll_interval 闸门)
        Puller->>FS: find_latest_ready()
        FS->>BMAC: 列 .ready cutoff
        alt cutoff 比上次新
            Puller->>FS: pull(pkl)
            FS->>BMAC: SFTP 下载
            FS-->>Puller: 原子写 data/remote_cache/
        end
    end
    alt market_pivot_* 有更新
        Puller->>SS: scan() (延迟 import)
        SS->>SS: 读 pivot 缓存 + normalize_pivot_symbol
        SS->>CMCdb: load_category_to_symbols()
        SS->>DB: DELETE 同 snapshot + INSERT 板块涨跌
    end
    Note over Sched,DB: cmc_bootstrap job 另起：启动+10s 检查 7天TTL，过期才刷 cmc_symbol_categories
