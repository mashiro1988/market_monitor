① 数据库表关系（erDiagram）

七张活跃表都没有数据库级 FK（SQLite 单文件 + WAL，靠应用层维护一致性）。下图虚线表示逻辑关联而非物理外键。所有 datetime 列均为 UTC naive。最后两张 cmc_symbol_categories / sector_returns 是 2026-05-17 板块管道新增。

    price_snapshots {
        int id PK

        float prev_price
        float change_pct
        float volume
        string source
        datetime created_at
    }
    news_items {
        int id PK

    }
    news_price_annotations {
        int id PK
        string symbol

        datetime context_end

② 一次扫描周期的数据流（sequenceDiagram）

从 run_scan_once 出发，价格 / 新闻 / 预测三条管道各自落库；告警评估读取所有三类记录后，再查 alert_logs 做冷却、写日志、推 WeCom。详细 file:line 见 DATAFLOW.md。

③ 一次 API 请求的数据流（sequenceDiagram · 以 /api/market/latest 为例）

所有 /api/* 请求结构相同：FastAPI 路由 → get_db() 依赖 → service 层查 DB / 算聚合 → Pydantic schema 序列化为 JSON。响应永远带双时间字段 timestamp_utc + timestamp_bj。
sequenceDiagram
    autonumber
    participant Browser as 浏览器
(React SPA)
    participant Client as frontend/src/api/client.ts
    participant Route as api/routes.py:market_latest
    participant Deps as api/deps.py:get_db
    participant Svc as market_service.get_latest_prices
    participant DB as price_snapshots

    Browser->>Client: 调用 api.marketLatest()
    Client->>Route: GET /api/market/latest
    Route->>Deps: Depends(get_db)
    Deps-->>Route: SQLAlchemy Session
    Route->>Svc: get_latest_prices(db)
    Svc->>DB: SELECT 每个 symbol 最新行
    DB-->>Svc: list[PriceSnapshot]
    Svc->>DB: SELECT 5m / 1h / 24h 历史
    DB-->>Svc: 历史快照
    Svc->>Svc: 计算涨跌幅
映射到 MarketLatestResponse
    Svc-->>Route: MarketLatestResponse
    Route-->>Client: JSON (含 timestamp_utc + timestamp_bj)
    Client-->>Browser: 渲染卡片

④ 板块远程数据流：BMAC pkl → sector_returns（sequenceDiagram）

远程数据周期：SFTP 拉 BMAC pivot pkl（numpy 2.x，靠 shim 加载）→ 归一化 symbol → 查 CMC 板块映射 → 等权聚合 → 写 sector_returns。榜单 API 读这张表，钻取 API 从 pivot 缓存现算。sector_scan 失败或写出 0 行时保留 pending，同一 cutoff 会在后续 cycle 重试。详见 DATAFLOW.md。
